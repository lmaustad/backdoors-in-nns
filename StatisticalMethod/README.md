# StatDet — Statistical Backdoor Detection

Statistical hypothesis testing for neural network backdoor detection.
Each test inspects weight distributions layer-by-layer, combines per-layer
p-values with Fisher's method, and flags a model if the combined p-value
falls below α = 0.05.

## Tests

| Name        | Score function    | Input    | What it measures |
|-------------|-------------------|----------|-----------------|
| `kurt`      | `score_kurtosis`  | flat `w` | Fisher kurtosis of weight distribution |
| `fft_peak`  | `score_fft_peak`  | flat `w` | Dominant non-DC frequency peak in sorted-weight spectrum |
| `zero`      | `zero_percentage` | flat `w` | Fraction of near-zero weights (\|w\| ≤ 0.0001) |
| `svd_spike` | `score_svd_spike` | tensor   | Ratio of top singular value to sum (spectrum concentration) |

All tests share the same inference pipeline:

1. Compute score per layer
2. Look up empirical p-value against null pool, matched by layer type (`conv` / `fc`)
3. Combine layer p-values with Fisher's method: `S = -2 Σ log(p_l)`
4. Look up empirical p-value of S against the null Fisher-S distribution
5. Flag model if `fisher_p ≤ α` (α = 0.05)

## Setup

### 1. Install dependencies

```bash
conda create -n backdoor python=3.10
conda activate backdoor
pip install -r requirements.txt
```

### 2. Set your paths

Edit [`config.py`](config.py) to point at your local model and dataset directories.
All scripts read paths from there — this is the only file you need to touch.

## Step-by-step workflow

### Step 1 — Train null models (skip if already on disk)

50 architectures × 3 datasets = 150 tier-1 null models.

```bash
cd ReferenceCollection

# Train one model (config_id 0–149):
python train_varied_cnn.py \
    --config_id 0 \
    --data_root /path/to/datasets \
    --out_dir   /path/to/models
```

Config ID → dataset mapping:
- IDs 0–49    → MNIST
- IDs 50–99   → FashionMNIST
- IDs 100–149 → GTSRB

Output: `{out_dir}/ckpt_{DATASET}/config_{id:03d}/model.pth`

### Step 2 — Build the null cache

```bash
python build_null.py
```

Loads the tier-1 null models from `config.NULL_DIRS`, computes per-layer score pools and
null Fisher-S distributions via leave-one-out, evaluates test accuracy, and writes:

- `null_cache.npz` — precomputed null distributions
- `null_model_configs.json` — per-model configs and accuracies
- `null_model_configs_table.tex` — LaTeX table

NPZ keys: `{test}_conv`, `{test}_fc`, `{test}_fisher_S` for each test in
`{kurt, fft_peak, zero, svd_spike}`.

Re-run whenever null models change.

**Optional:** cross-dataset LOO validation plots:

```bash
python build_null.py --validate
```

Accuracy summary for the current 150 tier-1 null models:

| Dataset      | n  | Mean acc | Range |
|--------------|----|----------|-------|
| MNIST        | 50 | 97.4%    | [96.4%, 98.7%] |
| FashionMNIST | 50 | 92.0%    | [90.5%, 92.6%] |
| GTSRB        | 50 | 90.1%    | [88.6%, 91.3%] |

### Step 3 — Run detection on backdoored model families

```bash
python StandardTesting.py --model dfba
python StandardTesting.py --model dfba_flex
python StandardTesting.py --model handcrafted
python StandardTesting.py --model handcrafted_flex

# Limit to first N seeds (default: 40):
python StandardTesting.py --model dfba --n 20
```

Outputs per model family: `out_{model}/results.json`, scatter plots, calibration plots.
Console prints TPR / FPR at α = 0.05 for each test.

## Checkpoint formats

`load_checkpoint` in `utils.py` handles:

- PyTorch `nn.Module` saved directly (`.pth` / `.pt` / `.ckpt`)
- Dict with `"state_dict"` or `"model"` key
- Raw `OrderedDict` state dict (all string keys)

Each tier-1 null model `.pth` is a dict `{"state_dict": ..., "config": {...}}`.
`config` contains: `conv_channels`, `fc_dims`, `kernel_size`, `optimizer`, `lr`,
`epochs`, `dataset`, `config_id`.
